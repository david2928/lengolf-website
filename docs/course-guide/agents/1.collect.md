# Stage 1: Collect

**Who does this:** Claude — triggered by you per region or batch

## What happens

Claude compiles structured raw data for each course in a given region.
No prose is written at this stage — only facts and structured fields.

For each course, Claude:
1. Uses web search to find the course's official website, green fees, contact details, and location
2. Cross-references with `knowledge/course-overrides.md` — any manual corrections there override web data
3. Fills every field in the schema (see `data/course-schema.md`) — missing fields are marked `null`, not guessed
4. Saves one JSON file per course to `data/1.raw/<region>/<slug>.json`

## How to trigger

```
"Collect Phase 1 Bangkok courses"
"Collect data for Nikanti Golf Club"
"Collect Hua Hin courses"
```

Claude will process every course listed under that region in `data/regions.md`.

## Data sources (in priority order)

| Source | Used when |
|---|---|
| `knowledge/course-overrides.md` | A manual correction exists for this field |
| Course official website | Fees, contact, facilities, designer |
| Web search | Fill gaps not on the official site |
| Claude knowledge | Last resort only — flagged as `"source": "model"` |

Any field sourced from Claude's training knowledge (not web search) is flagged so the verifier can prioritise it.
