# Stage 3: Verify

**Who does this:** Claude — runs automatically after Stage 2

## What happens

Claude runs verification across the entire region batch in one pass.
The output is a single batch report for you to read — not one report per course.

For each course, Claude verifies:

### 1. Factual fields (always web search)

These fields must be verified against a live source regardless of what Stage 1 found:

- Green fees (weekday and weekend)
- Caddie fee and cart fee
- Caddie required / cart required
- Distance from Bangkok and drive time
- Designer and year opened

If the Stage 1 value conflicts with what web search returns, it is flagged as a dispute in the batch report.

### 2. Prose claims (spot-check)

Claude re-reads the prose from Stage 2 and flags any claim that:
- Asserts a specific number or distance not in the `fields` object
- Uses a superlative ("best", "most scenic", "top-rated") without a cited source
- References a policy (dress code, booking rules) that should be verified

### 3. Schema field completeness

Every course must have values (non-null) for:
- `name`, `slug`, `region`, `province`
- `holes`, `par`
- `latitude`, `longitude`
- `green_fee_weekday_thb`, `green_fee_weekend_thb`
- `caddie_required`, `cart_required`
- `google_maps_url` (for sameAs schema)
- `website` (for sameAs schema)

Missing required fields are flagged as blockers — the course will not be published without them.

### 4. SEO check (per course)

- Title will be: `{Course Name} — Green Fees, Course Guide & Golf Club Rentals | Len.Golf`
- Meta description target: 140-160 characters
- Slug follows pattern: `<course-name>` (no region prefix — handled by URL structure)

## Output

```json

  "verification": {
    "status": "APPROVED" | "REVIEW_REQUIRED" | "BLOCKED",
    "verified_at": "2026-04-16",
    "disputes": [
      {
        "field": "green_fee_weekday_thb",
        "stage1_value": 2800,
        "web_search_found": 3200,
        "source_url": "https://...",
        "recommendation": "Update to 3,200 THB"
      }
    ],
    "missing_required_fields": [],
    "prose_flags": [
      {
        "section": "layout_and_experience",

        "issue": "Superlative without ranked source",

      "meta_description_ok": true

```
