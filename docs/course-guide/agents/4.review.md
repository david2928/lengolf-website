# Stage 4: Review (You)

**Who does this:** You — once per region batch

## What to read

One file:

```
docs/course-guide/reports/<region>-batch.report.md
```

This report contains only issues that need your input — disputes, missing data, prose flags.
Courses with no issues are listed as APPROVED in the summary table and need no action from you.

## Your options

**Everything looks good?**
> "Approve Bangkok batch"

Claude will:
1. Move all approved courses from `data/3.verified/` to `data/4.approved/`
2. Run Stage 5 (publish) for each approved course
3. Update the summary table in `reports/<region>-batch.report.md` with published status

**Fix a specific dispute:**
> "Fix Alpine green fee — weekday is 2,500 THB, weekend is 3,200 THB"
> "Nikanti caddie fee is 500 THB not 400 THB"

Claude will:
1. Update that course's JSON in `data/3.verified/`
2. Re-verify only that course (not the whole batch)
3. Update the batch report

**Accept a prose recommendation:**
> "Accept the recommendation for Riverdale"

Claude will update the prose and mark it resolved in the report.

**Ignore a non-blocking flag:**
> "Ignore the superlative flag for Riverdale — the wording is fine"

Claude notes it as accepted and moves on.

**Block a course from this batch:**
> "Hold Riverdale — I need to check the green fees myself"

Claude marks it as held and publishes the rest of the batch.

## What NOT to do

Don't manually edit the JSON files — tell Claude what to change so it re-verifies correctly.

## What you do NOT need to review

- The full page content for each course — the template handles layout, schema, and CTAs
- Individual course SEO — handled by the template and checked in Stage 3
- Any course that shows APPROVED in the summary table — these are ready to go
